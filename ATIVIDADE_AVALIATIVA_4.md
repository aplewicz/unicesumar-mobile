# Trabalho 4 - Aplicação de Lista de Tarefas com Gestão de Estado

**Valor: 2,5 pontos**

## Objetivo

Aplicar conceitos de **gestão de estado em Flutter** para desenvolver uma aplicação interativa.

## Descrição da Atividade

O grupo deverá desenvolver um aplicativo de **lista de tarefas (To-Do List)**.

O aplicativo deve permitir que o usuário gerencie tarefas, podendo criar, visualizar, marcar como concluídas e remover tarefas.

## Funcionalidades Mínimas

O aplicativo deve permitir:

- adicionar uma nova tarefa
- visualizar a lista de tarefas
- marcar uma tarefa como concluída
- remover uma tarefa.

## Requisitos Técnicos

O projeto deve obrigatoriamente utilizar:

- Flutter
- `ListView.builder`
- gestão de estado utilizando **Riverpod**.

O estado da aplicação deve ser utilizado para controlar:

- a lista de tarefas
- o status de conclusão das tarefas.

## Interface

A interface deve possuir:

- campo para inserir nova tarefa
- botão para adicionar tarefa
- lista exibindo todas as tarefas
- indicação visual de tarefas concluídas.

## Estrutura do Projeto

Espera-se que o projeto tenha organização mínima, como:

- widgets separados
- arquivo responsável pela gestão de estado
- separação entre interface e lógica.

## Documentação (README)

O README deve conter:

- descrição da aplicação
- explicação de como foi implementada a gestão de estado
- instruções para execução
- prints ou gifs demonstrando o funcionamento.
